# ローカルLLMプロンプトリレー 市場調査メモ v0.1

最終更新: 2026-04-27

## 1. 結論

このアプリには需要がある可能性がある。
ただし、結論から言うと、**いきなり巨大市場を取るタイプ** というより、
まずは **AIコーディングのヘビーユーザー向けの高密度なニッチ** から入るのが現実的である。

特に相性が良いのは、次の条件を持つ利用者である。

- Codex、Copilot、Cursor、Claude Code などを高頻度で使う
- 長文の依頼や長いログをよく送る
- 入力トークンや利用枠を気にしている
- クラウド前処理やAPIキー預かりに抵抗がある
- 圧縮よりも「品質を保って整える」ことに価値を感じる

現時点では、**単なるプロンプト短縮ツール** として出すと弱い。
一方で、
**「AIコーディングのコスト・品質・安全性を整える補助レイヤー」**
として出すなら、十分勝負になる。

## 2. 市場の大きさ

### 2.1 広い意味でのTAM

かなり広く見ると、AIを使う開発者市場はすでに非常に大きい。

- GitHubは、2025年時点で **180 million-plus developers** がGitHub上で活動していると公表している。  
  [GitHub Octoverse 2025](https://github.blog/news-insights/octoverse/octoverse-a-new-developer-joins-github-every-second-as-ai-leads-typescript-to-1/)

- Stack Overflowの2025 Developer Surveyでは、**84% of respondents are using or planning to use AI tools in their development process** とされている。  
  [Stack Overflow Developer Survey 2025 AI](https://survey.stackoverflow.co/2025/ai)

この2つをそのまま掛けるのは乱暴だが、
「AIを開発に使う層」はすでに極めて大きい、という判断は妥当である。

### 2.2 実際の狙い目となるSAM

このアプリにとって本当に意味があるのは、
**AIをたまに試す人** ではなく、
**コスト・上限・長文コンテキストに日常的に悩む人** である。

ここで参考になるのは、実際のAIコーディングツールの有料利用や継続利用の規模である。

- OpenAIは2026年4月2日に、**more than 2 million builders now use Codex every week** と公表している。  
  さらに **more than 9 million paying business users rely on ChatGPT for work** とも述べている。  
  [OpenAI: Codex now offers pay-as-you-go pricing for teams](https://openai.com/index/codex-flexible-pricing-for-teams/)

- GitHub Copilotは、2025年7月時点で **20 million** の累計ユーザーに達したと報じられている。  
  [TechCrunch: GitHub Copilot crosses 20M all-time users](https://techcrunch.com/2025/07/30/github-copilot-crosses-20-million-users/)

- GitHub Copilotの有料購読者数については、Microsoftの四半期決算に基づく報道で、2026年初頭に **4.7 million paid subscribers** とされている。これは公式GitHub Docsではなく報道ベースである。  
  [iTWire: GitHub Copilot booms, smashing 4.7 million subscriber milestone](https://itwire.com/business-it-news/enterprise-solutions/github-copilot-booms-smashing-4-7-million-subscriber-milestone.html)

このアプリの初期SAMは、かなり現実的に見積もって
**数百万〜1,000万弱規模の「AIコーディング高頻度ユーザー」**
と考えるのが妥当である。

## 3. 需要がある理由

### 3.1 AIコーディング利用はもう主流

Stack Overflowの2025 Surveyでは、
**51% of professional developers use AI tools daily** とされている。  
  [Stack Overflow Developer Survey 2025 AI](https://survey.stackoverflow.co/2025/ai)

つまり、AIコーディングは「試してみる段階」から、
日常的な作業の一部へ移りつつある。

### 3.2 しかし、信頼と品質には不満がある

同じSurveyでは、
**46% がAIの出力を distrust、33% が trust** であり、
強く信頼している人はごく少数である。  
  [Stack Overflow Developer Survey 2025](https://survey.stackoverflow.co/2025)

これは、このアプリにとって重要である。
なぜなら、あなた様の案は
「何でも自動で短くする」ではなく、
**保持条件と欠落リスクを見せながら送信前に整える**
方向だからである。

### 3.3 トークンベース課金と利用上限が前面に出てきた

OpenAIはCodexのチーム向け料金で、2026年4月から
**token consumption** ベースの pay-as-you-go を前面に出した。  
  [OpenAI: Codex now offers pay-as-you-go pricing for teams](https://openai.com/index/codex-flexible-pricing-for-teams/)

GitHub Copilot、Cursor、Codex、Windsurf なども、
単純な「無制限」ではなく、上限、premium requests、usage tiers を持つ。

つまり、今後は
**「AIを使えるか」より「どのくらい効率よく使えるか」**
が価値になりやすい。

## 4. 類似・競合プロダクト

### 4.1 直接競合に近いもの

#### Terse

- macOS / iOS向けの token optimizer
- Claude Code、Cursor、ChatGPT、Copilot などを対象
- サイト上では **40-70%** のコスト削減を主張
- ローカル実行、agent session monitoring、prompt compression を前面に出している  
  [Terse](https://www.terseai.org/)

補足:

- VS Code Marketplace上のTerse拡張は、2026-04-27時点で **8 installs** と表示されていた  
  [Terse VS Code Marketplace](https://marketplace.visualstudio.com/items?itemName=LucasZeng.terse-optimizer)

この数字は製品全体の利用者数ではないが、
少なくとも **このカテゴリがまだ初期市場** であることを示す一つのシグナルにはなる。

#### Pith

- APIプロキシ型の prompt optimization
- 「saved amount に応じて課金」という価格設計
- Free / Pro があり、**$5-$6/mo + savings share** を打ち出している  
  [Pith](https://pithtoken.ai/)

特徴:

- provider-agnostic
- original forwarding fallback
- 料金は「削減額から払う」思想

ただし、公開されている利用者数や売上の実績は見当たらない。

#### PromptShrink

- ローカルで動くシンプルな prompt compression ツール
- no servers / 100% local & private を訴求  
  [PromptShrink](https://www.promptshrink.com/)

特徴:

- 非常に軽量
- ただし、文法語の除去中心で、Codex用途の保持条件チェックまでは弱い

こちらも公開されている売上や利用実績は見当たらない。

## 5. 市場性の判断

### 5.1 良い点

- AIコーディング利用者数そのものは増えている
- トークン課金・利用枠への意識は強まっている
- 品質不安と検証需要がある
- ローカル完結・APIキー不要は明確な差別化になる
- Codex / Cursor / Claude Code のような高頻度利用シーンと相性がよい

### 5.2 厳しい点

- 「プロンプト圧縮だけ」のカテゴリはまだ大きく立ち上がっていない
- 大手本体製品に吸収されやすい
- 補助ツールなので、主役プロダクトより価格上限が低い
- 効果が数値で見えないと継続課金されにくい

## 6. 売上見込み

### 6.1 前提

このアプリは、Copilot / Cursor / Codex のような本体製品ではなく、
それらを補助する **companion utility** である。

そのため、価格帯は本体より低めが自然である。

現実的な価格帯の目安:

- 個人: **$6-$12 / month**
- 小規模チーム: **$10-$20 / user / month**
- 企業: カスタム

この価格帯は、次の競合価格の下に置くと収まりが良い。

- GitHub Copilot Pro: $10/mo  
  [GitHub Copilot plans](https://docs.github.com/en/copilot/get-started/plans)
- Cursor Pro: $20/mo  
  [Cursor pricing](https://cursor.com/en-US/pricing)
- Cursor Teams: $40/user/mo  
  [Cursor pricing](https://cursor.com/en-US/pricing)
- Pith Pro: $5-$6/mo + savings share  
  [Pith](https://pithtoken.ai/)

### 6.2 かなり保守的な見込み

前提:

- 個人 1,500人
- チーム 25社
- 1社あたり20席
- 個人ARPU $8
- チームARPU $12

概算:

- 個人MRR = 1,500 x $8 = **$12,000**
- チームMRR = 500 x $12 = **$6,000**
- 合計MRR = **$18,000**
- 合計ARR = **$216,000**

これは、十分ありえる初期SaaS規模である。

### 6.3 現実的なベースケース

前提:

- 個人 6,000人
- チーム 100社
- 1社あたり25席
- 個人ARPU $8
- チームARPU $12

概算:

- 個人MRR = 6,000 x $8 = **$48,000**
- チームMRR = 2,500 x $12 = **$30,000**
- 合計MRR = **$78,000**
- 合計ARR = **$936,000**

このレンジに届けば、**かなり良いニッチSaaS** と言える。

### 6.4 強い実行ができた場合のアップサイド

前提:

- 個人 18,000人
- チーム 250社
- 1社あたり30席
- 個人ARPU $8
- チームARPU $12

概算:

- 個人MRR = 18,000 x $8 = **$144,000**
- チームMRR = 7,500 x $12 = **$90,000**
- 合計MRR = **$234,000**
- 合計ARR = **$2.8M**

この規模は可能性としてはあるが、
そのためには単なる圧縮ツールではなく、
**コスト可視化、比較、チーム導入、品質保証** まで広げる必要がある。

## 7. どのくらいの売上が「現実的」か

率直に言うと、
現時点のプロダクト像のままなら、

- **$100k-$300k ARR**: 十分現実的
- **$500k-$1M ARR**: 良い実行が必要だが狙える
- **$2M+ ARR**: プロダクトの拡張と強い流通が必要
- **Cursor級 / Copilot級**: そのままでは難しい

という見立てである。

理由は単純で、
このアプリは **主役のAI開発環境ではなく、補助レイヤー** だからである。

## 8. 売上を伸ばすために必要なこと

単なる「トークン圧縮」では弱い。
売上を伸ばすには、以下の価値が必要である。

1. **コスト可視化**
   - いくら削減したかを1回 / 月次 / 年次で見せる

2. **品質保証**
   - 重要条件保持、欠落リスク、fallback を見せる

3. **Codex特化**
   - 一般用途ではなく、開発依頼向けに最適化する

4. **チーム導入**
   - profile共有、利用分析、標準ルール

5. **ローカル安心感**
   - APIキー不要、外部送信なし、端末内前処理

## 9. おすすめの事業判断

このアプリは、
**大規模VC型の一本勝負** というより、
**収益化しやすい高密度ニッチSaaS** として見るのがよい。

最初の勝ち筋:

- Codexユーザーに絞る
- 個人課金から始める
- コスト削減額を可視化する
- 品質劣化しないことを証明する

その後の伸ばし方:

- Cursor / Claude Code / Copilotにも広げる
- チーム管理機能を足す
- コストダッシュボード化する

別案として残しておく販路:

- AIを前面に出すノートPCメーカーやAI PC陣営へのOEM / ISV連携
- Intel / AMD / Qualcomm のAI PC開発者プログラム経由での展開
- Lenovo / HP / Dell などの開発者向け同梱、紹介、法人提案での展開

この案は、直販より立ち上がりが遅い一方で、
通れば単価と流通規模を大きくしやすい。
ただし、最初から主戦場にするより、
**直販で実績を作った後の第2段階の販路**
として扱うのが現実的である。

## 10. 最終判断

この市場には需要がある。
ただし、需要が大きいのは「AIコーディング市場全体」であって、
「プロンプト圧縮だけ」の市場はまだ立ち上がり途中である。

したがって、このアプリは

**プロンプトを短くするツール**
ではなく、
**AIコーディングのコスト・品質・安全性を整えるローカル制御レイヤー**

として売り出すのが最も筋が良い。
