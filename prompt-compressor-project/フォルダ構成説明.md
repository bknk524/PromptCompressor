# フォルダ構成説明

このファイルは、Prompt Compressor プロジェクト内の各フォルダが何を担当するかをまとめたものです。

## ソフト本体

### `applications/`

アプリケーションの入口を置くフォルダです。

- `command-line-compressor/`: コマンドラインからプロンプト圧縮を実行するCLI
- `local-web-compressor-ui/`: ブラウザで使うローカルWeb UI
- `codex-mcp-compressor-server/`: Codexから呼び出すためのMCPサーバー雛形
- `desktop-app-notes/`: 将来のデスクトップアプリ化に向けたメモ

### `libraries/`

複数の入口から共通利用する中核ロジックを置くフォルダです。

- `compression-core/`: 圧縮リクエスト、実行サービス、ランタイム接続、検証処理など

### `settings/`

プロンプト圧縮で使うモデル、実行環境、プロファイル、ポリシー設定を置くフォルダです。

- `compression-profiles.yaml`: UIやCLIで選ぶプロファイル一覧
- `model-catalog.yaml`: llama.cppやLM Studioで使うモデル参照
- `runtime-backends.yaml`: llama.cppやLM Studioなどの実行先設定
- `compression-policies/`: 圧縮時に守るルール

### `prompt-templates/`

LLMへ渡す圧縮用プロンプトテンプレートを置くフォルダです。

### `json-schemas/`

圧縮リクエストと圧縮結果のJSON形式を定義するフォルダです。

## ローカル実行用

### `local-runtime-binaries/`

ローカル実行に使う外部バイナリを置くフォルダです。

例: `llama-cli.exe`

### `local-model-files/`

ローカルLLMのモデルファイルを置くフォルダです。

例: `.gguf` 形式のモデルファイル

## 企画・管理資料

### `企画資料/`

ソフトの企画、要件、設計、調査、進捗をまとめる資料フォルダです。

- `企画書_ローカルプロンプト圧縮.md`: 企画書本文
- `要件定義_ローカルプロンプト圧縮.md`: 要件定義
- `システム設計_ローカルプロンプト圧縮.md`: システム設計
- `市場調査_ローカルプロンプト圧縮.md`: 市場調査
- `進捗状況.md`: 現在の進捗
- `作業リスト.md`: 次にやる作業
- `Codex用プロジェクト参照.md`: Codexが作業前に読む参照資料
- `Codex起動プロンプト.md`: Codex作業開始用のメモ

### `評価メモ/`

将来的な評価ケースやテスト観点をまとめるフォルダです。

## 生成物・削除してよいもの

### `target/`

Rust/Cargoのビルド生成物です。

削除しても問題ありません。`cargo build` や `cargo check` を実行すると再生成されます。

## ルート直下の主なファイル

- `README.md`: プロジェクト概要と起動方法
- `Cargo.toml`: Rust workspace設定
- `Cargo.lock`: 依存関係の固定情報
- `rust-toolchain.toml`: 使用するRust toolchain指定
- `AGENTS.md`: Codex向けの作業ルール
- `.gitignore`: Git管理から除外するファイル設定
- `.editorconfig`: エディタ共通設定
- `フォルダ構成説明.md`: この説明ファイル
