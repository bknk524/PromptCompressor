# フォルダ構成

このプロジェクトは、実行に必要なものを `application/`、企画や評価など実行に直接関係しない資料を `資料/` に分けています。

```text
PromptCompressor/
  application/
    core/
      compression-core/
    ui/
      web/
    interfaces/
      cli/
      mcp/
    config/
    resources/
      prompts/
      schemas/
    local/
      models/
      runtimes/
      cache/
      logs/
      state/
  資料/
    企画資料/
    評価メモ/
    将来構想/
    フォルダ構成.md
```

## `application/core/`

`compression-core/` は UI、CLI、MCP に共通する圧縮エンジンです。内部では次の責務に分けています。

- `compression/`: 圧縮サービス、トークン推定、出力検証。
- `config/`: プロファイル設定の読み込み。
- `runtime/`: LM Studio、llama.cpp、内部サーバーの実行処理。
- `types.rs` と `error.rs`: 全体で共有するデータ形式とエラー。

## `application/ui/`

`web/` はブラウザ UI の HTTP サーバーと静的ファイルを置く場所です。圧縮ロジックやモデルのファイルパスを直接持たず、プロファイルを通じてコアへ依頼します。

## `application/interfaces/`

- `cli/`: ターミナルで利用する入口。
- `mcp/`: Codex から利用する MCP の入口。

どちらも `core/` を呼ぶ薄い接続層です。圧縮ルールは持ちません。

## `application/config/`

Git 管理する共有設定です。

- `model-catalog.yaml`: モデルの能力、テンプレート、ローカル配置先。
- `compression-profiles.yaml`: UI と CLI で選ぶプロファイルとフォールバック。
- `runtime-backends.yaml`: LM Studio と llama.cpp の接続・起動設定。
- `compression-policies/`: 圧縮強度と保持条件。
- `user/`: 個別 PC 用の将来の上書き設定。内容は Git 管理しません。

## `application/resources/`

- `prompts/`: ローカル LLM へ渡す圧縮テンプレート。
- `schemas/`: 圧縮要求と圧縮結果の JSON スキーマ。

## `application/local/`

PC 固有の実行データです。モデル選択の定義は `config/` に置き、実ファイルと実行状態だけをここへ保存します。

- `models/`: 内部実行で使う GGUF モデル。
- `runtimes/`: `llama-cli.exe` と `llama-server.exe`。
- `cache/`: 再生成できるキャッシュ。
- `logs/`: ローカル診断ログ。
- `state/`: 将来のモデル在庫と実行状態。

`models/`、`runtimes/`、`cache/`、`logs/`、`state/` の実データは Git 管理しません。

## `資料/`

`企画資料/`、`評価メモ/`、`将来構想/` はアプリの起動に不要な資料です。実行コードと混ざらないよう `application/` の外へ置いています。

## 起動

リポジトリ直下で次を実行してください。設定は `application/config/` から自動で検出されます。

```powershell
cargo run -p prompt-compressor-local-ui -- --host 127.0.0.1 --port 8787
```
